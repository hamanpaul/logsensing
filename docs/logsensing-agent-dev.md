# **系統日誌自動化分析與 AI 診斷工具 \- 開發計畫書**

## **1\. 專案概述**

本專案旨在開發一套全自動化的日誌分析 CLI 工具，專門針對設備進行上千次 Power Cycle（重開機）壓力測試時所產生的巨量、多執行緒交錯且帶有時間軸偏移的日誌進行解析。系統將結合 Drain3 動態日誌探勘演算法與 LLM Agent 技術，將除錯流程從傳統的「人工搜尋」升級為「AI 自動根因診斷」。

## **2\. 核心架構設計 (三階段管線 Pipeline)**

### **Phase 1: 資料清洗與動態建模 (Parser & Normalizer)**

* **串流切檔 (Stream Splitter)：** 以 Bootloader 特定字串為錨點，將巨型實體檔案切割為單一 Boot Cycle 記憶體區塊，避免 OOM (Out of Memory)。  
* **Drain3 動態探勘：** 放棄靜態 Regex，導入 Drain3 演算法。系統將自動剝離常數模板與動態參數（如 Hex Address、IP），適應韌體改版造成的未知日誌格式。  
* **頻道分流 (Demultiplexing)：** 依據萃取出的 PID 或模組前綴，將 File System、Network 等多執行緒干擾訊息拆分至獨立虛擬頻道。

### **Phase 2: 邏輯判定與異常擷取 (Rule Engine & Analyzer)**

* **時間軸基準線 (Baseline Profiling)：** 運算正常開機各里程碑的平均時間差 (Delta Time)。  
* **異常捕捉與裁切：** 套用時間容忍閥值與致命錯誤（如 Kernel panic）規則庫。精準命中錯誤後，僅裁切案發現場前後 50 行乾淨日誌。  
* **OTel 標準化輸出：** 捨棄自定義 JSON，採用 OpenTelemetry (OTel) 標準，為每次開機與模組初始化賦予 TraceID 與 SpanID，最終匯出極度輕量的 anomalies.json。

### **Phase 3: AI 診斷與互動助理 (Agent CLI)**

* **自動根因分析 (Automated RCA)：** 讀取 anomalies.json，由 LLM 自動為每一次失敗的 Cycle 撰寫人類易讀的除錯摘要。  
* **互動式問答：** 提供終端機互動介面，允許開發者透過自然語言下達指令（例如：agent analyze cycle 120）。  
* **混合 RAG 檢索 (選配)：** 結合 BM25 精準匹配與 FAISS 語意檢索，讓 Agent 能查閱硬體規格書與歷史 Jira Tickets 提供修復建議。

## ---

**4\. 預期效益與未來擴充**

* **大幅降低維護技術債：** Drain3 的自適應能力讓團隊在面對頻繁的韌體迭代時，無需不斷回頭修改解析規則。  
* **精準定位時間異常：** 有效解決多執行緒環境下，特定硬體模組初始化逾時（Timeout）被海量雜訊淹沒的痛點。  
* **跨領域生態系整合：** 這套解耦架構不僅適用於單一網通產品的壓力測試，其高度可擴充的特性，也非常適合直接移植到需要長期監控設備健康度與系統穩定性的智慧家庭整合專案（如 HomeSensing）中，作為底層的自動化維運守護行程。
